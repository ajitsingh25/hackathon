Vulnerability :    Broken Authentication

Severity:           HIGH

Description: 

1. It is a vulnerability in the session management that exposes the application to varied risks.
2. If the session tokens are not properly protected, an attacker can hijack an active session and assume the identity of a user.

Remediation: Define a stringent security policy around the below aspects.
1. Password Strength
2. Number of login attempts
3. Password changing procedure.
4. How the passwords would be stored.
5. Change cookie after authentication
6. Secured transmissions of passwords using SSL.
