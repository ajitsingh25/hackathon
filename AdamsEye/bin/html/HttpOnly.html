Vulnerability :    HTTPOnly

Severity:           HIGH

Description: 

1. HttpOnly is an additional flag included in a Set-Cookie HTTP response header
2. Using the HttpOnly flag when generating a cookie helps mitigate the risk of client side script accessing the protected cookie (if the browser supports it)

Remediation:

Set the HttpOnly cookie attribute for the sessionCookie. This can be set as below.
1. Set the HttpOnly cookie programmitically
2. Set it for SessionCookie in the Application Server descriptor files
