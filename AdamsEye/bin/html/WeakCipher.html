Vulnerability :    Weak Ciphers

Severity:           HIGH

Description: 

1. Occurs if the HTTP protocol is used to transmit sensitive information (e.g. credentials transmitted over HTTP). HTTP protocol does not ensure confidentiality of the data.

2. When the SSL/TLS service is present it is good but it increments the attack surface and the following vulnerabilities exist:

  • SSL/TLS protocols, ciphers, keys and renegotiation must be properly configured.
  • Certificate validity must be ensured.

Remediation:

1. Weak ciphers must not be used; no NULL ciphers suite, due to no encryption used; no Anonymous Diffie-Hellmann, due to not provides authentication.

2. Weak protocols must be disabled.

3. Renegotiation must be properly configured

4. X.509 certificates key length must be strong (e.g. if RSA or DSA is used the key must be at least 1024 bits).

5. X.509 certificates must be signed only with secure hashing algoritms (e.g. not signed using MD5 hash, due to known collision attacks on this hash).
