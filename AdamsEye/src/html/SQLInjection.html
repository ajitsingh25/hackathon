Vulnerability :    SQL Injection

Severity:           HIGH

Description: 

1. Insertion or "injection" of a SQL query via the input data from the client to the application.
2. Can exploit and read , modify sensitive data from the database, execute administration operations on the database (such as shutdown the DBMS), recover the content of a given file present on the DBMS file system and in some cases issue commands to the operating system.

Causes:

1. Data enters a program from an untrusted source.
2. The data used to dynamically construct a SQL query

Remediation:

1. Parameterized queries using bound, typed parameters
2. Careful use of parameterized stored procedures.
